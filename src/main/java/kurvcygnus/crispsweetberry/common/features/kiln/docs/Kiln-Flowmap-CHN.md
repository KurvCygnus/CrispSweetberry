# 窑炉流程图

🌏 | [English Version](Kiln-Flowmap.md)

## 介绍

本文档由Kurv撰写, 通过把窑炉本身复杂的运作机制拆分成两大板块, 并可视化来解释窑炉的实现原理.

***在你完全理解窑炉的运作原理之前, 我们不建议修改任何窑炉相关的代码, 即使那一部分看着很简单.***

## 窑炉总处理处理流程图

**窑炉本身通过组件化的方式实现, 其中 <u>[`KilnProgressModel`](../blockstates/components/KilnProgressModel.java)</u> 负责记录进度, 而 <u>[`KilnProgressCalculator`](../blockstates/components/KilnProgressCalculator.java)</u> 则只关心逻辑计算.**

关于**全流程总览**的大纲, 请自行翻阅<u>[`KilnBlockEntity#serverTick`](../blockstates/KilnBlockEntity.java)</u>以了解. `serverTick` 本身已经足够清晰, 无需流程图.

此外, **<u>[状态推导(`#deduceProcessState`)](../blockstates/KilnBlockEntity.java)</u> 也不会有介绍**, 该方法相当直观, 同样无需介绍.

### 进度计算

*提示:*
*本流程隶属于 <u>[`KilnProgressCalculator#clculateRates`](../blockstates/components/KilnProgressCalculator.java)</u> 方法.*

#### 初始化

```mermaid
graph TB;
    subgraph 计算初始化检查
        %% 节点声明
        初始化开始[开始配方的遍历];
        初始化-非法检查[检查配方的类型是否合法];
        跳越到异常处理;
        转到正常处理["转到正常初始化"];

        %% 流程
        初始化开始 --> 初始化-非法检查;

        初始化-非法检查 --> |"emptyRecipe & tipRecipe"| 跳越到异常处理;
        初始化-非法检查 --> |<code>KilnRecipe</code>| 转到正常处理;
    end
    
    后记[注意: 步骤'计算初始化检查'的遍历与步骤'正常配方处理'的遍历是同一个循环,<br>由于更新流程图会导致这两个步骤变得及其复杂, 因此留后记比修改更好.];
    
    subgraph 异常配方类型处理 
        %% 节点声明
        初始化-异常-通用-预处理[结束遍历];
        初始化-异常-通用-处理["将值<code>NORMAL_PROGRESS_RATE</code>赋给变量<code>lastProcessFactor</code>,<br>并结束平衡状态以防止数据污染"];
        初始化-异常-通用-返回["返回双进度减少了<code>NORMAL_PROGRESS_RATE</code>百分比的结果(行为与<code>COOLDOWN</code>对齐)"];
        初始化-异常-返回-分支(获取非法配方的具体类型);
        初始化-异常-返回-非法["将返回的<code>ResultType</code>的值敲定为<code>INVALID</code>"];
        初始化-异常-返回-提示["将返回的<code>ResultType</code>的值敲定为<code>BLAST_TIP</code>"];
        跳过[跳过剩下的计算];
        
        %% 流程
        跳越到异常处理 --> 初始化-异常-通用-预处理 --> 初始化-异常-通用-处理 --> 初始化-异常-通用-返回 --> 初始化-异常-返回-分支;
        
        初始化-异常-返回-分支 --> |emptyRecipe| 初始化-异常-返回-非法 --> 跳过;
        初始化-异常-返回-分支 --> |tipRecipe| 初始化-异常-返回-提示 --> 跳过;
    end
    
    subgraph 正常配方处理 
        %% 节点声明
        初始化-正常-分支("配方的<code>processFactor</code>是否小于标准值(<code>STANDARD_PROCESS_FACTOR</code>)?");
        初始化-正常-数值坍塌-检查[使索引值增加1];
        初始化-正常-数值坍塌-策略[使用取均值的策略];
        初始化-正常-默认-策略[使用相乘策略];
        初始化完成["确认正确的<code>processFactor</code>"];
        
        %% 流程
        转到正常处理 --> 初始化-正常-分支;
        
        初始化-正常-分支 --> |更小| 初始化-正常-数值坍塌-检查 --> 初始化-正常-分支;
        
        初始化-正常-分支 --> |所有配方都更小| 初始化-正常-数值坍塌-策略 --> 初始化完成;
        初始化-正常-分支 --> |任何一个配方更大| 初始化-正常-默认-策略 --> 初始化完成;
    end
```

#### 计算

```mermaid
graph TD;
    subgraph 计算模式推导 
        %% 节点声明
        计算-模式确认("当前计算模式是否为 <code>BALANCING</code>?");

        计算-默认行为[计算进度增加比];
        计算-平衡-预处理["将 <code>isBalancing</code> 标记设为 <code>true</code>"];
        计算-平衡-处理[开始调整视觉进度与实际进度];
        计算-平衡-返回["返回包含平衡后进度的结果,<br>并将 <code>ResultType</code> 设为 <code>BALANCING</code>,<br><code>Trend</code>的值取决于速率的正负"]
        计算-默认-预处理[计算当前 Tick 的进度率];
        计算-默认-检查["检查 <code>processState</code> 的具体类型"];
        计算-默认-冷却分支[将进度率转为负值];
        计算-默认-返回["将进度率累加进进度并返回结果,<br><code>ResultType</code> 的值取决于 <code>processState</code>,<br><code>Trend</code>的值取决于速率的正负"];
        
        %% 流程
        计算-模式确认 --> |是| 计算-平衡-处理;
        计算-模式确认 --> |否| 处理因子比较;
        
        处理因子比较 --> |否| 计算-平衡-预处理 --> 计算-平衡-处理;
        处理因子比较 --> |是| 计算-默认行为 --> 计算-默认-预处理 --> 计算-默认-检查;
        
        计算-默认-检查 --> |<code>WORKING</code>| 计算-默认-返回;
        计算-默认-检查 --> |<code>COOLDOWN</code>| 计算-默认-冷却分支 --> 计算-默认-返回;
        计算-平衡-处理 --> 计算-平衡-返回;
    end
```

### 产物产出处理

进度更新之前的处理部分比较直观, 此处不再赘述.

```mermaid
graph TD;
    subgraph 进度检查 
        %% 节点声明
        检查("进度是否大于 1D (100%)?");
        
        %% 流程
        检查 --> |是| 继续流程;
        检查 --> |否| 跳过;
    end
    
    subgraph 处理预检查 
        %% 节点声明
        遍历-开始[开始遍历输入槽];
        
        %% 虽然逻辑上可能不必要, 但为了应对极端情况依然保留
        遍历-检查("当前遍历的物品是否可处理?");
        遍历-非法分支["终止遍历, 返回 <code>false</code> (不更新数据)"];
        遍历-合法-处理[收集该输入的产物];
        遍历-合法-分支("遍历是否结束?");
        遍历-继续[移动到下一个槽位];
        遍历-完成[预检查遍历完成];
        转到模拟准备[进入模拟插入前的准备阶段];
        
        %% 流程
        继续流程 --> 遍历-开始 --> 遍历-检查;
        
        遍历-检查 --> |否| 遍历-非法分支;
        遍历-检查 --> |是| 遍历-合法-处理 --> 遍历-合法-分支;
        
        遍历-合法-分支 --> |否| 遍历-继续 --> 遍历-检查;
        遍历-合法-分支 --> |是| 遍历-完成;
    end
    
    遍历-完成 --> 转到模拟准备;
    
    subgraph 模拟插入准备
    %% 节点声明
        复制输出槽["复制输出槽内容到模拟结果中<br>(用于模拟插入)"];
        开始模拟[使用复制的输出槽内容启动模拟];
        模拟-遍历开始[启动插入模拟遍历];
        
        %% 流程
        转到模拟准备 --> 复制输出槽 --> 开始模拟 --> 模拟-遍历开始;
    end
    
    subgraph 模拟插入逻辑 
        %% 节点声明
        模拟-检查("产物是否能与当前遍历的<br>输出槽合并?");
        模拟-空内容检查("该槽位是否是第一个空槽?");
        模拟-空内容检查-真[记录该空槽索引];
        模拟-非法-处理[标记本次模拟尝试无效];
        模拟-模拟槽位-推进[移动到下一个输出槽];
        模拟-遍历-检查("该产物是否已遍历完所有输出槽?");
        模拟-遍历-结束-分支("是否所有合并尝试都无效?");
        模拟-遍历-结束-非法分支-分支("是否存在记录的空槽?");
        模拟-遍历-结束-非法-预期情况[将产物合并至记录的空槽];
        模拟-遍历-结束-合法-融合[将产物合并至对应的输出槽];
        模拟失败["模拟失败, 跳过后续处理,<br>返回 <code>false</code>"];
        模拟-遍历-结束-检查("所有输入的产物是否都已模拟完毕?");
        模拟-遍历-推进[处理下一个输入的产物];
        模拟-遍历-结束["完全终止模拟, 返回 <code>true</code><br>以应用模拟结果"];
        
        %% 流程
        模拟-遍历开始 --> 模拟-检查;
        
        模拟-检查 --> |否| 模拟-非法-处理 --> 模拟-空内容检查;
        模拟-检查 --> |是| 模拟-空内容检查;
        
        模拟-空内容检查 --> |是| 模拟-空内容检查-真 --> 模拟-遍历-检查;
        模拟-空内容检查 --> |否| 模拟-遍历-检查;
        
        模拟-遍历-检查 --> |否| 模拟-模拟槽位-推进 --> 模拟-检查;
        模拟-遍历-检查 --> |是| 模拟-遍历-结束-分支;
        
        模拟-遍历-结束-分支 --> |否| 模拟-遍历-结束-合法-融合 --> 模拟-遍历-结束-检查;
        模拟-遍历-结束-分支 --> |是| 模拟-遍历-结束-非法分支-分支;
        
        模拟-遍历-结束-非法分支-分支 --> |是| 模拟-遍历-结束-非法-预期情况 --> 模拟-遍历-结束-检查;
        模拟-遍历-结束-非法分支-分支 --> |否| 模拟失败;
        
        模拟-遍历-结束-检查 --> |否| 模拟-遍历-推进 --> 模拟-检查;
        模拟-遍历-结束-检查 --> |是| 模拟-遍历-结束;
    end
```
